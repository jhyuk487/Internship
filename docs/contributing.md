# 기여 가이드

이 프로젝트는 로컬 개발 중심의 간단한 워크플로를 사용합니다.

## 작업 원칙
- 변경 범위를 작게 유지합니다.
- 동작이 바뀌면 문서를 함께 업데이트합니다.
- 시크릿/`.env`는 커밋하지 않습니다.

## 브랜치
- `feature/chat-history`, `fix/login-flow`처럼 짧고 의미 있는 이름을 사용합니다.

## 테스트
- 자동 테스트는 아직 없습니다.
- 변경 후 `docs/test_checklist.md`로 수동 검증합니다.

## 코드 스타일
- Python: 명확한 함수 분리와 명시적 이름을 선호합니다.
- JavaScript: 큰 inline DOM 문자열은 가급적 헬퍼 함수로 분리합니다.

## 리뷰 체크리스트
- API 응답 스키마가 변경되었는가?
- 보호 라우트가 여전히 보호되는가?
- 게스트 플로우가 정상인가?
- 문서가 최신 상태인가?
